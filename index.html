🗄️ Configuration Supabase requise

Si la table user_roles n'existe pas encore, exécutez ce SQL dans Supabase → SQL Editor :

CREATE TABLE IF NOT EXISTS user_roles (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  email text UNIQUE NOT NULL,
  role text NOT NULL DEFAULT 'user',
  magasins_ids text[] DEFAULT '{}',
  permissions text[] DEFAULT '{}',
  note text DEFAULT '',
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
ALTER TABLE user_roles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "auth_read" ON user_roles FOR SELECT TO authenticated USING (true);
CREATE POLICY "auth_write" ON user_roles FOR ALL TO authenticated USING (true) WITH CHECK (true);
⚠️ Réinitialisation du stock à zéro

Supprime tous les mouvements (entrées, sorties, transferts) et remet les quantités initiales à 0.
Les produits, fournisseurs et commandes sont conservés.
⚠️ Cette action est irréversible.

📋 Conseil : Exportez d'abord un rapport PDF (inventaire + mouvements) avant de réinitialiser.
🚨 Suppression totale des données

Supprime DÉFINITIVEMENT tous les produits, mouvements, fournisseurs et commandes.
Les paramètres de configuration sont conservés.
🚨 Cette action est totalement irréversible.

🛑 Utiliser uniquement pour repartir sur une base vierge. Toutes les données opérationnelles seront perdues.
